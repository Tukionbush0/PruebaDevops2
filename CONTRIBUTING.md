# Contributing


## Flujo de trabajo (GitFlow simplificado)


1. `main` → rama protegida con la versión estable.
2. Para desarrollar nuevas funcionalidades:
- Crea una rama desde `main`: `feature/<descripcion-corta>`
- Haz commits pequeños y con mensajes claros (imperativo): `feat: agregar endpoint /users`.
3. Cuando termines, abre un Pull Request hacia `main` y pide revisión.
4. Después de aprobación, merge con `Squash and merge` para mantener historial claro.
